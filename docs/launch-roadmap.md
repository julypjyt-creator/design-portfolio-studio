# 网站可上线路线图（面向当前项目）

## 0. 当前状态

- 前台、后台、统计页面：已完成可用基础版
- Next.js：已升级到 `15.5.15`（安全审计通过）
- 数据存储：当前使用浏览器本地存储（`localStorage`）
- 结论：可演示，但还不适合正式对外长期运营

## 1. 上线前必须项（P0）

### 1.1 部署到公网（1-2 天）

- 将代码托管到 GitHub
- 接入 Vercel 并完成首次部署
- 绑定正式域名（如 `portfolio.xxx.com`）
- 开启 HTTPS（Vercel 默认）

### 1.2 统一云端数据（2-4 天）

- 引入 MySQL（你现有可用）
- 使用 Prisma 建模：`works`, `categories`, `tags`, `work_tags`
- 将本地 mock/localStorage 迁移到数据库读写
- 后台“新增/编辑/删除/筛选”全部改为服务端持久化

### 1.3 图片云存储（1-2 天）

- 接入 OSS / S3 / Cloudinary 任一方案
- 上传后存 URL 到数据库，而不是存 base64
- 增加图片大小与格式校验

### 1.4 管理员登录（1-2 天）

- 接入 Auth.js / NextAuth
- 先做单管理员账号密码登录
- 后台页面增加鉴权保护（未登录不能访问）

## 2. 上线增强项（P1）

### 2.1 数据导入导出（1 天）

- 保留现有 CSV 导出
- 增加 CSV 批量导入（可选）

### 2.2 运维与监控（1 天）

- 接入错误监控（如 Sentry）
- 配置日志与告警

### 2.3 SEO 与分享优化（0.5-1 天）

- 配置 sitemap / robots
- 作品详情页补充 OG 图与 meta

## 3. 正式上线验收清单

- [ ] 公网域名可访问
- [ ] 后台需要登录才能进入
- [ ] 新增作品后，前台可见且刷新不丢失
- [ ] 图片加载稳定，移动端正常
- [ ] 统计页数据与后台列表一致
- [ ] 安全审计无高危/严重漏洞
- [ ] 可执行数据备份（数据库 + 图片）
